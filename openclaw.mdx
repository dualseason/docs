OpenClaw 是一个自托管 AI 助手网关。它可以把 Telegram、飞书、Discord、WhatsApp 等消息入口统一接到同一个 AI 后端。对 Xcompute 来说，它最适合作为“多渠道助手入口”。

## 官方安装方式

如果你还没准备 Node 环境，先看 [环境准备：NVM、Node.js 22 和 npm](/node-setup)。

```bash
npm install -g openclaw
```

安装后验证：

```bash
openclaw --version
```

<Note>
  OpenClaw 官方更推荐 `Node 24`，但 `Node 22.16+` 仍然兼容。如果你当前整个文档站统一基于 `Node 22`，也可以先跑通。
</Note>

## 接入 Xcompute 的推荐方式

官方推荐先运行 `onboard` 向导。对 Xcompute 来说，最实用的做法是选择 OpenAI 兼容或自定义 provider，然后把基础地址切到 Xcompute。

<Steps>
  <Step title="运行初始化向导">
    ```bash
    openclaw onboard
    ```
  </Step>

  <Step title="选择 OpenAI 兼容或自定义 provider">
    在向导里，把模型提供方切到支持自定义 `Base URL` 的方式。
  </Step>

  <Step title="填写 Xcompute 信息">
    使用下面这组值：

    - `Base URL`: `https://xcompute.us/v1`
    - `API Key`: 你的 Xcompute API Key
    - `Model`: 你要使用的模型 ID，例如 `gpt-5` 或 `claude-sonnet-4-5-20250929`
  </Step>

  <Step title="打开控制台验证">
    ```bash
    openclaw dashboard
    ```

    控制台打开后，发一条测试消息确认是否已能正常调用 Xcompute。
  </Step>
</Steps>

## 日常使用教程

### 1. 查看网关状态

```bash
openclaw gateway status
```

### 2. 打开控制台

```bash
openclaw dashboard
```

### 3. 配置消息入口

最常见的是 Telegram、飞书、Discord。建议先从一个最简单的渠道开始，不要一上来同时接很多渠道。

### 4. 用专用密钥隔离渠道调用

建议给 OpenClaw 单独分配一把 API Key，避免和应用服务、Claude Code、OpenCode 共用。

## 什么时候适合用 OpenClaw

1. 你想把 AI 助手接到 Telegram、飞书、Discord 等聊天入口
2. 你想在自托管环境里统一管理消息入口和模型调用
3. 你希望不同渠道共用同一套 Xcompute 模型出口

<Tip>
  建议给 OpenClaw 单独分配一个 API Key，不要和应用服务、Claude Code、OpenCode 共用。
</Tip>
