## 核心概念

OpenCode 有三种交互模式，理解它们有助于更好地使用：

| 模式 | 说明 | 适用场景 |
| --- | --- | --- |
| **Agent** | 自主编码模式，可调用工具读写文件、执行命令 | 需要 AI 自动完成编码任务时 |
| **Plan** | 计划模式，AI 会分析需求并给出实施方案 | 复杂功能开发前先出方案 |
| **Ask** | 问答模式，纯对话，不调用工具 | 代码审查、知识问答、技术讨论 |

切换方式：在会话中输入 `/agent`、`/plan`、`/ask` 即可切换。

### Sub-agent（子代理）

OpenCode 内部使用多种专门的子代理来处理不同任务：

- **Codex**：核心编程代理，负责代码编写、调试、重构
- **Planner**：负责分解复杂任务、制定执行计划
- **Browser**：负责网页浏览和信息收集
- 以及其他专用代理

这些子代理由 OpenCode 自动管理和调度，无需手动配置。如果你为某个模型配置了 `maxTokens` 参数，确保该值足够大，以便子代理能完成复杂任务。

如果你还没准备 Node 环境，先看 [环境准备：NVM、Node.js 22 和 npm](/node-setup)。

  </Tab>

  <Tab title="官方安装脚本">

  </Tab>

  <Tab title="Homebrew 安装">

安装后验证：

```bash
opencode --version
```

如果能看到版本号，就说明安装阶段已经完成。

### 安装后提示「command not found」？

npm 全局安装的路径可能不在 `PATH` 中。运行以下命令查看并修正：

```bash
# 查看 npm 全局 bin 路径
npm bin -g

# 将其加入 shell 配置（以 bash 为例）
echo 'export PATH="$(npm bin -g):$PATH"' >> ~/.bashrc
source ~/.bashrc
```

## 配置 Xcompute

推荐使用全局配置文件：`~/.config/opencode/opencode.json`

```json
{
  "$schema": "https://opencode.ai/config.json",
  "provider": {
    "xcompute": {
      "npm": "@ai-sdk/openai-compatible",
      "name": "Xcompute",
      "options": {
        "baseURL": "https://xcompute.us/v1",
        "apiKey": "{env:XCOMPUTE_API_KEY}"
      },
      "models": {
        "gpt-5.6-sol": {
          "name": "GPT 5.6 Sol"
        },
        "gpt-5.5": {
          "name": "GPT 5.5"
        },
        "claude-sonnet-5": {
          "name": "Claude Sonnet 5"
        },
        "claude-opus-4-8": {
          "name": "Claude Opus 4.8"
        },
        "gemini-3.5-flash": {
          "name": "Gemini 3.5 Flash"
        },
        "gemini-3.1-pro-high": {
          "name": "Gemini 3.1 Pro High"
        },
        "deepseek-v4-flash": {
          "name": "DeepSeek V4 Flash"
        }
      }
    }
  },
  "model": "xcompute/gpt-5.6-sol"
}
```

### 生产环境配置示例

针对团队协作或多项目管理场景，推荐这种更完善的配置：

```json
{
  "$schema": "https://opencode.ai/config.json",
  "provider": {
    "xcompute": {
      "npm": "@ai-sdk/openai-compatible",
      "name": "Xcompute",
      "options": {
        "baseURL": "https://xcompute.us/v1",
        "apiKey": "{env:XCOMPUTE_API_KEY}"
      },
      "models": {
        "gpt-5.6-sol": {
          "name": "GPT 5.6 Sol",
          "maxTokens": 65536
        },
        "gpt-5.5": {
          "name": "GPT 5.5",
          "maxTokens": 32768
        },
        "claude-sonnet-5": {
          "name": "Claude Sonnet 5",
          "maxTokens": 65536
        },
        "grok-4.3": {
          "name": "Grok 4.3",
          "maxTokens": 65536
        },
        "deepseek-v4-flash": {
          "name": "DeepSeek V4 Flash",
          "maxTokens": 65536
        }
      }
    }
  },
  "model": "xcompute/gpt-5.6-sol",
  "permissions": {
    "mode": "bySession",
    "rules": [
      {
        "match": "src/**/*.ts",
        "allow": ["read", "write", "edit"]
      },
      {
        "match": "package.json",
        "allow": ["read", "write"]
      }
    ]
  }
}
```

### 配置文件放在哪里

推荐放在：

- macOS / Linux：`~/.config/opencode/opencode.json`
- Windows：用户目录对应的 `opencode` 配置目录

## 或者，直接用全局环境变量

```bash
export XCOMPUTE_API_KEY="YOUR_XCOMPUTE_API_KEY"
opencode
```

如果你只是先想测试一下是否能接通，用这种方式最快。

### 环境变量支持

- `OPENCODE_API_KEY`：作用于所有 provider
- 自定义变量名（如 `XCOMPUTE_API_KEY`）：在配置中用 `{env:变量名}` 引用
- `OPENCODE_MODEL`：指定默认模型

## 初始化项目

官方推荐在项目里第一次启动后执行：

```text
/init
```

它会分析项目并生成 `AGENTS.md`，给 AI 一份项目指南。后续会话中 OpenCode 会自动读取 `AGENTS.md` 作为系统提示，帮助它理解你的代码风格和项目规范。

### AGENTS.md 应该包含什么

- **项目概述**：做什么的、技术栈、目录结构
- **开发规范**：命名风格、测试要求、commit 格式
- **编码约定**：状态管理方案、API 风格、组件设计模式
- **构建测试命令**：测试、构建、lint 配置
- **风格偏好**：代码风格、注释规范、UI 组件库等

**注意事项：**

- 使用纯文本或 Markdown 格式，`/init` 会自动生成初版
- OpenCode 还支持 `AGENTS.local.md`（本地覆盖，不提交到 Git）和 `AGENTS.custom.md`（用户级全局参考）
- 如果项目下没有 `AGENTS.md`，OpenCode 会使用 `.opencode/instructions.md`

### 解释仓库

```text
How is authentication handled in this repo?
```

### 做功能前先出计划

OpenCode 官方文档推荐先切到计划模式，再讨论实现方案。

```text
先给我一个实现方案，再开始改代码
```

### 让它直接改代码

```text
请为设置页增加一个登录校验，并复用 notes 路由里的实现方式
```

### 回滚不满意的改动

```text
/undo
```

### 提交规范的 commit

结合 OpenCode 与 Git 工作流，推荐的工作流程：

```text
1. /plan 分析需要修改的文件
2. AI 给出方案后，确认执行
3. 修改完成后，运行测试：/run npm test
4. 提交代码：/run git add -A && git commit -m "feat: xxx"
5. 如果有 lint：/run npm run lint --fix
```

## MCP 服务器配置（进阶）

OpenCode 支持通过 MCP（Model Context Protocol）协议连接外部工具，获得更强能力。MCP 服务器在 `~/.config/opencode/opencode.json` 中配置：

```json
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/path/to/allowed/dir"]
    },
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_TOKEN": "<YOUR_GITHUB_TOKEN>"
      }
    },
    "playwright": {
      "command": "npx",
      "args": ["-y", "@playwright/mcp"]
    },
    "sequential-thinking": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-sequential-thinking"]
    }
  }
}
```

### 常用 MCP 服务器

| 服务器 | 用途 | 启动命令 |
| --- | --- | --- |
| Filesystem | 受控的文件读写访问 | `npx -y @modelcontextprotocol/server-filesystem <path>` |
| GitHub | PR、Issue、代码搜索 | `npx -y @modelcontextprotocol/server-github` |
| Playwright | 浏览器自动化操作 | `npx -y @playwright/mcp` |
| Sequential Thinking | 深度推理和多步思考 | `npx -y @modelcontextprotocol/server-sequential-thinking` |

## 配置文件完整参考

| 配置项 | 类型 | 说明 | 必填 |
| --- | --- | --- | --- |
| `$schema` | string | 配置文件 JSON Schema 地址 | 否 |
| `model` | string | 默认使用的模型，格式 `provider/modelId` | 是 |
| `provider` | object | 模型提供商配置 | 是 |
| `provider.<name>.npm` | string | SDK npm 包名 | 是 |
| `provider.<name>.name` | string | 提供商显示名称 | 否 |
| `provider.<name>.options.baseURL` | string | API 地址 | 是 |
| `provider.<name>.options.apiKey` | string | API Key，支持 `{env:VAR}` 引用环境变量 | 是 |
| `provider.<name>.models` | object | 模型列表，key 为模型 ID，value 包含 `name` 和可选的 `maxTokens` | 是 |
| `permissions.mode` | string | 权限模式：`acceptAll` / `byTool` / `bySession` | 否 |
| `permissions.rules` | array | 自定义文件路径权限规则 | 否 |
| `mcpServers` | object | MCP 外部工具服务配置 | 否 |

## OpenCode vs ChatGPT

| 维度 | OpenCode | ChatGPT（桌面版） |
| --- | --- | --- |
| API 接入 | 支持自定义 `baseURL` 和 provider | 仅官方 API，无法中转 |
| 使用方式 | CLI / IDE 扩展 / 桌面 | 桌面应用 / 网页 |
| 文件读写 | 原生支持，自动读写项目文件 | 需要手动上传或复制粘贴 |
| 终端执行 | `/run` 命令直接在项目终端执行 | 不支持 |
| 配置文件 | JSON 配置，精细控制模型、权限、MCP | 无配置文件 |
| 多模型管理 | 一个配置多个模型，随时切换 | 需要切换对话 |
| 适用场景 | 编程开发、代码重构、项目维护 | 日常问答、文档写作 |

## 常见问题

### 为什么 OpenCode 很适合接中转站？

因为它本身就支持自定义 provider 和 `baseURL`，比很多只支持官方 API 的客户端更灵活。

### 为什么推荐先执行 `/init`？

因为它会自动帮你生成项目级说明文件 `AGENTS.md`，后续会话中 AI 能更准确地理解你的代码风格和项目结构。

### 如何指定默认模型？

```json
"model": "xcompute/gpt-5.6-sol"
```

或通过环境变量：

```bash
export OPENCODE_MODEL="xcompute/gpt-5.6-sol"
```

### 如何切换模型？

在会话中直接输入 `/model` 查看可选模型列表，然后输入 `/model xcompute/gpt-5.5` 即可切换。

### 提示「insufficient balance」？

Xcompute 账户余额不足，去 [控制台](https://xcompute.us/dashboard/overview) 充值。

### 提示「model not found」？

检查配置中的模型 ID 是否正确。完整模型列表请参考 [模型定价](/api-reference/models) 页面。

### 提示 API Key 错误？

检查 `~/.config/opencode/opencode.json` 中的 `apiKey` 配置，或确认环境变量 `XCOMPUTE_API_KEY` 已正确设置。注意：环境变量设置后需要重新启动 opencode 才能生效。

### 如何升级 OpenCode？

```bash
npm update -g opencode-ai
```

### 如何同时管理多个项目的配置？

在项目目录下创建 `.opencode/opencode.json`，OpenCode 会自动合并全局和项目级配置。项目级配置只需要设置 `model`、`permissions` 等项目相关字段即可。

### 如何在 IDE 中使用？

- **VS Code**：在扩展市场搜索 "OpenCode" 安装
- **JetBrains**：在插件市场搜索 "OpenCode" 安装

IDE 扩展共享 `~/.config/opencode/opencode.json`，无需额外配置即可使用 Xcompute。

### 安装后提示「command not found」？

npm 全局安装路径可能不在 `PATH` 中。运行 `npm bin -g` 查看路径并加入 shell 配置。

### MCP 服务器连接失败？

检查 MCP 服务器的 `command` 和 `args` 配置是否正确，以及相关依赖是否已安装。确保网络能访问 MCP 服务器的资源。

## 什么时候适合用 OpenCode

1. 你想在同一套代码 Agent 工作流中切换不同模型。
2. 你希望把多个开发工具的调用都统一到一个平台管理。
3. 你想通过配置文件精细控制 provider、model 和权限策略。
