3. 表单、Webhook、数据库、消息通知联动
4. AI 内容处理流水线

## 官方安装方式

如果你想快速体验，官方推荐可以先用：

```bash
npx n8n
```

如果你想长期跑本地实例，也可以全局安装：

```bash
npm install -g n8n
n8n
```

## 对非技术用户更推荐哪种安装方式

如果你只是测试：

1. 优先 `npx n8n`

如果你想长期运行：

1. 再考虑全局安装或 Docker

默认访问地址：

```text
http://localhost:5678
```

## 第一步：先确认平台能打开

你成功的标志是：

1. 浏览器里能打开 `http://localhost:5678`
2. 能进入编辑器界面
3. 能新建工作流

## 接入 Xcompute 的两种方式

### 方式一：用 OpenAI 节点

n8n 自带 OpenAI 节点。对于支持自定义 `Base URL` 的版本，你可以直接把它接到 Xcompute：

- `Base URL`: `https://xcompute.us/v1`
- `API Key`: 你的 Xcompute API Key

这种方式适合：

1. 你已经在用 OpenAI 节点
2. 你的节点版本支持自定义基础地址

### 方式二：用 HTTP Request 节点

如果你当前版本的 OpenAI 节点不方便填自定义地址，最稳的方案就是直接用 `HTTP Request` 节点调用 Xcompute。

例如：

```bash
POST https://xcompute.us/v1/chat/completions
```

请求体：

```json
{
  "model": "gpt-5",
  "messages": [
    {
      "role": "user",
      "content": "请总结今天新增的工单内容"
    }
  ]
}
```

这种方式最通用，也最容易排错。

## 非技术用户推荐顺序

最建议按下面顺序来：

1. 打开 n8n
2. 新建一个最简单工作流
3. 先用 `HTTP Request` 节点请求 Xcompute
4. 先让它返回一句文本
5. 跑通后，再接 Webhook、表单、数据库、消息通知

## 推荐工作流

1. `Webhook` 或业务触发器进入
2. `Set` / `Code` 节点整理上下文
3. `HTTP Request` 或 `OpenAI` 节点请求 Xcompute
4. 解析返回结果
5. 发回飞书、Telegram、邮件、数据库或下一条业务链路

## 第一次上手建议

### 最简单测试流

1. 新建工作流
2. 放一个手动触发器
3. 放一个 `HTTP Request` 节点
4. 请求 `https://xcompute.us/v1/chat/completions`
5. 返回一条简单回复
6. 点运行

如果能看到模型返回内容，说明你已经接通成功。

## 常见问题

### 为什么推荐先用 HTTP Request 节点？

因为它最直接、最透明。非技术用户一旦遇到问题，也更容易把请求体和报错发给别人排查。

### 什么时候再用 OpenAI 节点？

当你已经确认：

1. Xcompute 可用
2. 请求格式没问题
3. 只是想把节点体验做得更方便

这时候再切到 OpenAI 节点更合适。

## 什么时候适合用 n8n

1. 你要做 AI 自动化流程
2. 你需要把 AI 输出接到很多第三方系统
3. 你希望非纯工程团队也能看懂流程图
