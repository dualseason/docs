## 对接 Xcompute API

### 第一步：获取 Xcompute API Key

1. 注册并登录 [Xcompute](https://xcompute.us)
2. 进入 [API 密钥页面](https://xcompute.us/keys)
3. 点击 **创建新密钥**
4. 根据要使用的模型选择渠道分组
5. 复制以 `sk-` 开头的 API Key

| 使用场景 | 建议渠道 |
| --- | --- |
| GPT 模型 | GPT 福利渠道 0.9 折 |
| Claude 日常使用 | Kiro 渠道 1.8 折 |
| Claude 高质量或蒸馏需求 | 官 Key 渠道 4.9 折 |

<Warning>
  API Key 相当于账号密码。请勿公开、分享或提交到代码仓库。
</Warning>

### 第二步：添加供应商

1. 打开 Kelivo
2. 进入 **设置** > **供应商管理**
3. 点击 **添加供应商**
4. 选择 OpenAI 兼容的供应商类型
5. 填写名称、Base URL 和 API Key

| 字段 | 值 |
| --- | --- |
| **名称** | `Xcompute`，也可以自定义 |
| **Base URL** | `https://xcompute.us/v1` |
| **API Key** | Xcompute API Key |

国内访问国际站时，可以使用加速入口：

```text
https://intl.dualseason.com/v1
```

### 第三步：添加模型

**Claude 系列：**

| 模型名称 | 模型 ID | 说明 |
| --- | --- | --- |
| Claude Fable 5 | `claude-fable-5` | 高强度推理和复杂任务 |
| Claude Sonnet 5 | `claude-sonnet-5` | 质量与成本平衡 |
| Claude Opus 4.8 | `claude-opus-4-8` | Claude Opus 工作流 |

**GPT 系列：**

| 模型名称 | 模型 ID | 说明 |
| --- | --- | --- |
| GPT 5.6 Sol | `gpt-5.6-sol` | 高能力，复杂推理和代码生成 |
| GPT 5.6 Terra | `gpt-5.6-terra` | 通用模型，日常对话 |
| GPT 5.6 Luna | `gpt-5.6-luna` | 轻量模型，低成本快速响应 |
| GPT 5.5 | `gpt-5.5` | 复杂对话和 Agent 任务 |
| GPT 5.4 | `gpt-5.4` | 日常对话和成本控制 |

### 第四步：开始对话

1. 返回聊天主页并新建对话
2. 点击模型选择器
3. 选择刚添加的 Xcompute 模型
4. 输入消息并发送

## MCP 工具调用

Kelivo 是少数在移动端支持 MCP（Model Context Protocol）的客户端。

### 内置 MCP 工具

自带 fetch 工具，无需额外配置，AI 可自动获取网页内容。

在助手编辑页面的「可用工具」中开启即可。

### 连接外部 MCP 服务器

进入 **设置** → **MCP 服务器** → **+** 添加，填入服务器地址和认证信息即可。

### 移动端使用场景

- 外出时让 AI 联网查资料
- 结合文档分析处理 PDF/Word
- 连接代码 MCP 工具在手机上 debug

## 自定义助手

Kelivo 支持创建多个 AI 助手，每个助手可绑定不同模型、系统提示词和工具。

### 创建步骤

1. 打开侧边栏，点击 **默认助手** 区域
2. 点击 **+** 创建新助手
3. 配置以下项：

| 配置项 | 说明 |
| --- | --- |
| 助手名称 | 如"编程助手"、"翻译官" |
| 默认模型 | 此助手默认使用的模型 |
| 温度 | 0=严谨，1=发散 |
| 系统提示词 | 定义角色和行为 |
| 可用工具 | MCP 工具开关 |

### 推荐配置

**编程助手：** 模型 `claude-sonnet-5`，温度 0.3
**写作助手：** 模型 `gpt-5.5`，温度 0.7
**翻译助手：** 模型 `gpt-5.4`，温度 0.2

## 常见问题

### API Key 保存在哪里？

供应商配置保存在设备本地。启用远程备份或通过二维码导出配置时，应确认备份和二维码中是否包含密钥。

### 国内网络能使用吗？

可以。将 Base URL 设置为 `https://intl.dualseason.com/v1`，通过加速入口访问国际站。

### 支持 HarmonyOS 吗？

支持。HarmonyOS 版本由 [kelivo-ohos](https://github.com/Chevey339/kelivo-ohos) 项目单独维护。

### 如何切换模型？

在对话界面点击当前模型名称，然后选择已配置的模型。

### 支持后台生成吗？

Android 版本支持后台生成，可以在设置中启用。

### 请求失败如何排查？

1. 确认 Base URL 包含 `/v1`
2. 确认 API Key 正确、未过期且账户有余额
3. 确认模型 ID 与 Xcompute 可用模型一致
4. 国内网络尝试切换到加速入口
5. 暂时关闭联网搜索或 MCP 工具，排除外部服务错误
