跟着这份指南，你可以先理解 Xcompute 的工作方式，再完成首次接入。

## 先理解 Xcompute 在做什么

Xcompute 的核心能力，是把多种 AI 模型统一到一套调用方式里。你不需要分别对接不同厂商，也不需要为每个工具维护不同配置。

<Info>
  如果你的应用已经兼容 OpenAI 接口协议，通常只需要替换 `Base URL` 和 `API Key`，就可以开始迁移。
</Info>

## 三步完成上手

<Steps>
  <Step title="创建 API Key">
    登录控制台，进入 API Key 页面创建密钥。建议按业务或团队拆分不同密钥，方便后续监控与管理。
  </Step>
  <Step title="选择模型与渠道">
    根据任务目标选择模型，并结合渠道分组与计费方式控制成本和稳定性。
  </Step>
  <Step title="在工具或代码中完成配置">
    将目标工具中的接口地址切换到 Xcompute，然后填入你的 API Key。
  </Step>
</Steps>

## 你需要关注的两个概念

### 渠道分组

- `官方渠道`：更适合稳定性要求高的生产任务。
- `特惠渠道`：更适合对价格敏感的测试、实验或批量任务。
- `default`：系统自动优选，适合新手直接开始使用。

### 计费模式

- `按量计费`：适合短对话、轻量调用和高频问答。
- `按次计费`：适合长文生成、复杂分析和长上下文任务。

<Tip>
  如果单次请求和回复的总长度很长，按次计费通常更容易控制预算。
</Tip>

## 常见问题

<AccordionGroup>
  <Accordion title="我应该先从哪种方式开始？">
    如果你是第一次接触平台，先创建 API Key，再用在线调试或站内聊天页面验证效果，最后再接入自己的应用。
  </Accordion>
  <Accordion title="如何查看使用量和费用？">
    你可以在控制台查看调用记录、用量和费用明细。
  </Accordion>
  <Accordion title="可以给不同业务分开密钥吗？">
    可以。建议按应用、团队或环境拆分 API Key，便于限额管理和问题排查。
  </Accordion>
</AccordionGroup>
