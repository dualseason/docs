这一页是整个集成教程的基础篇。你只要先把 `NVM`、`Node.js 22` 和 `npm` 安装好，后面接入 `Claude Code`、`Codex`、`OpenCode` 就会简单很多。

</Info>

## 这篇教程适合谁

如果你属于下面任意一种情况，就建议先看这篇：

1. 你还没装过 `Node.js`
2. 你分不清 `Node.js`、`npm`、`nvm` 是什么
3. 你需要在一台电脑上安装多个 AI 编程工具
4. 你希望后面教程里的命令可以直接复制执行

## 你最终会得到什么

做完这篇以后，你会拥有：

1. 一个可以切换 Node 版本的环境
2. 一个稳定的 `Node.js 22` 运行时
3. 一个可用的 `npm`
4. 三个可直接运行的工具：`Claude Code`、`Codex`、`OpenCode`

## 为什么先装 NVM

推荐先装 `NVM`，原因很简单：

1. 你后面可能会同时碰到 `Node 20`、`Node 22`、`Node 24` 的项目。
2. `Claude Code`、`Codex`、`OpenCode` 这些 CLI 更新很快，切 Node 版本会比重装省事很多。
3. Windows 和 macOS / Linux 的安装方式不同，提前统一思路会少踩坑。

## 第一步：安装 NVM

<Tabs>
  <Tab title="macOS / Linux">
    如果你是 `Mac` 或 `Linux` 用户，推荐直接使用官方安装脚本。

    ```bash
    curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.4/install.sh | bash
    ```

    安装完成后，重新打开终端。也可以手动加载：

    ```bash
    source ~/.bashrc
    ```

    如果你使用的是 `zsh`，改成：

    ```bash
    source ~/.zshrc
    ```

    然后验证：

    ```bash
    command -v nvm
    ```

    如果终端输出 `nvm`，说明安装成功。
  </Tab>

  <Tab title="Windows">
    Windows 不用官方 POSIX 版 `nvm`，而是使用 `nvm-windows`。

    下载地址：

    - [https://github.com/coreybutler/nvm-windows/releases](https://github.com/coreybutler/nvm-windows/releases)

    你只需要：

    1. 打开下载页
    2. 下载最新的安装包
    3. 双击安装
    4. 安装完成后重开 PowerShell

    安装完成后，打开 PowerShell，先验证：

    ```powershell
    nvm version
    ```

    如果能看到版本号，再继续下一步。
  </Tab>
</Tabs>

## 第二步：安装 Node.js 22

<Info>
  这里统一推荐 `Node.js 22`。原因是它足够新、兼容性也好，适合作为这批 CLI 工具的统一基础环境。
</Info>

<Warning>
  如果你的机器里之前已经装过系统级 Node.js，理论上也能继续用，但更容易出现版本冲突。非技术用户优先走 `NVM + Node 22` 最省事。
</Warning>

<Tabs>
  <Tab title="macOS / Linux">
    执行下面三条命令：

    ```bash
    nvm install 22
    nvm use 22
    nvm alias default 22
    ```

    然后验证：

    ```bash
    node -v
    npm -v
    ```

    正常情况下你会看到：

    1. `node -v` 返回 `v22.x.x`
    2. `npm -v` 返回一个 npm 版本号
  </Tab>

  <Tab title="Windows">
    执行下面命令：

    ```powershell
    nvm install 22.22.2
    nvm use 22.22.2
    ```

    然后验证：

    ```powershell
    node -v
    npm -v
    ```

    如果命令找不到，请先完全关闭 PowerShell 再重新打开一次。
  </Tab>
</Tabs>

### NVM 常用命令速查

安装好 NVM 后，以下命令会经常用到：

| 命令 | 说明 | 示例 |
| --- | --- | --- |
| `nvm install <version>` | 安装指定版本 | `nvm install 22` |
| `nvm use <version>` | 切换到指定版本 | `nvm use 22` |
| `nvm ls` / `nvm list` | 列出已安装版本 | `nvm ls` |
| `nvm ls-remote` | 列出可安装版本 | `nvm ls-remote` |
| `nvm alias default <version>` | 设置默认版本 | `nvm alias default 22` |
| `nvm uninstall <version>` | 卸载版本 | `nvm uninstall 20` |
| `nvm current` | 查看当前版本 | `nvm current` |

### 配置 npm 镜像源（国内用户推荐）

如果你在国内访问 npm 官方源较慢，可以切换到淘宝镜像：

```bash
npm config set registry https://registry.npmmirror.com
npm config get registry   # 验证
```

如需切换回官方源：

```bash
npm config set registry https://registry.npmjs.org
```

### 配置代理（如需）

如果你需要通过 HTTP 代理访问网络：

```bash
npm config set proxy http://127.0.0.1:7890
npm config set https-proxy http://127.0.0.1:7890
```

NVM 安装 Node.js 时也支持镜像下载：

```bash
# macOS / Linux
export NVM_NODEJS_ORG_MIRROR=https://npmmirror.com/mirrors/node

# Windows (nvm-windows)
nvm node_mirror https://npmmirror.com/mirrors/node/
```

## 一键环境验证

执行以下命令快速确认环境是否就绪：

**macOS / Linux：**

```bash
echo "NVM: $(nvm --version 2>/dev/null || echo '未安装')"
echo "Node: $(node -v 2>/dev/null || echo '未安装')"
echo "npm: $(npm -v 2>/dev/null || echo '未安装')"
echo "镜像源: $(npm config get registry 2>/dev/null || echo '未知')"
```

**Windows PowerShell：**

```powershell
Write-Host "NVM: $(nvm version 2>$null)"
Write-Host "Node: $(node -v 2>$null)"
Write-Host "npm: $(npm -v 2>$null)"
Write-Host "镜像源: $(npm config get registry 2>$null)"
```

## 常见问题

### 安装完 `nvm` 后提示 `command not found`

通常是 shell 配置还没重新加载。

- `bash`：`source ~/.bashrc`
- `zsh`：`source ~/.zshrc`
- Windows：重开一个 PowerShell 窗口

如果仍然找不到，检查配置文件末尾是否包含以下内容：

```bash
export NVM_DIR="$HOME/.nvm"
[ -s "$NVM_DIR/nvm.sh" ] && \. "$NVM_DIR/nvm.sh"
```

### 切换 Node 版本后 `node -v` 还是旧版本

- Windows：**必须以管理员身份运行** PowerShell，nvm-windows 需要管理员权限创建符号链接
- macOS/Linux：检查 `which node` 是否指向 NVM 管理目录（`~/.nvm/versions/...`），如果指向 `/usr/local/bin/node`，说明有其他 Node 安装冲突，卸载即可
- 检查 PATH 中是否有其他 Node.js 路径优先级更高

### npm 提示权限不足（EACCES）

优先确认你是在 `nvm` 管理的 Node 环境里执行：

```bash
which node   # 应输出 ~/.nvm/versions/node/...
```

NVM 将 Node 安装在用户目录下，一般不需要 `sudo`。如果仍然提示权限问题：

```bash
npm config set prefix "$HOME/.npm-global"
# 然后将 $HOME/.npm-global/bin 加入 PATH
```

不要默认上来就加 `sudo npm install -g ...`。

### npm install 卡住或超时

1. 切换到国内镜像源：`npm config set registry https://registry.npmmirror.com`
2. 清除缓存重试：`npm cache clean --force && npm install`
3. 增加超时时间：`npm config set fetch-timeout 120000`

### Windows 下 nvm install 下载失败

配置 nvm-windows 的下载镜像：

```powershell
nvm node_mirror https://npmmirror.com/mirrors/node/
nvm npm_mirror https://npmmirror.com/mirrors/npm/
```

### Windows PowerShell 执行策略问题

如果提示"禁止运行脚本"，以管理员身份执行：

```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```
