## 工具简介

CC Switch 是一个图形化的 AI 工具配置管理器，主要功能包括：

- **一键切换**：快速切换不同 AI 工具的供应商配置
- **多工具管理**：统一管理 Claude Code、Claude Desktop、Codex、Gemini CLI、OpenCode、OpenClaw 和 Hermes Agent
- **速度测试**：测试不同 API 端点的响应速度
- **配置备份**：自动备份配置并支持恢复
- **MCP 管理**：统一配置和同步 MCP 服务器

## 获取 Xcompute API Key

1. 登录 [Xcompute 控制台](https://xcompute.us)
2. 进入 [API 密钥页面](https://xcompute.us/keys)
3. 点击 **创建新密钥**
4. 根据要使用的模型选择对应渠道分组
5. 复制生成的 API Key（以 `sk-` 开头）

| 使用场景 | 建议渠道 |
| --- | --- |
| GPT 模型 | GPT 福利渠道 0.9 折 |
| Claude 日常使用 | Kiro 渠道 1.8 折 |
| Claude 高质量或蒸馏需求 | 官 Key 渠道 4.9 折 |

<Warning>
  API Key 相当于账号密码。请勿公开、分享或提交到代码仓库。
</Warning>

## 配置 Xcompute 供应商

### 第一步：选择 AI 工具

启动 CC Switch，在顶部工具栏选择要配置的工具。本教程以 Claude Code 为例。

<Frame>
  ![在 CC Switch 中选择 AI 工具](/images/image-9.png)
</Frame>

### 第二步：添加供应商

点击右上角的 **添加** 按钮，创建自定义供应商。已经添加的国际站和国内站会显示在供应商列表中。

<Frame>
  ![CC Switch 供应商列表](/images/image-8.png)
</Frame>

### 第三步：填写基础配置

供应商名称可以自定义。API Key 填写从 Xcompute 获取的 `sk-` 密钥，请求地址需要根据工具选择。

| 工具 | 请求地址 | API 格式 |
| --- | --- | --- |
| Claude Code | `https://xcompute.us` | Anthropic Messages（原生） |
| Codex | `https://xcompute.us/v1` | OpenAI |

如果你在国内访问国际站，可以将域名替换为 `intl.dualseason.com`：

| 工具 | 加速入口请求地址 |
| --- | --- |
| Claude Code | `https://intl.dualseason.com` |
| Codex | `https://intl.dualseason.com/v1` |

<Warning>
  Claude Code 的请求地址不要添加 `/v1`。Codex 使用兼容 OpenAI 的接口，需要保留 `/v1`。
</Warning>

<Frame>
  ![填写 Xcompute 供应商配置](/images/image-10.png)
</Frame>

### 第四步：配置 Claude 模型映射

使用 Claude Code 时，按下表填写模型映射：

| 模型角色 | 显示名称 | 实际请求模型 |
| --- | --- | --- |
| Sonnet | `claude-sonnet-5` | `claude-sonnet-5` |
| Opus | `claude-opus-4-8` | `claude-opus-4-8` |
| Fable | `claude-fable-5` | `claude-fable-5` |

默认兜底模型可以根据你的渠道和使用习惯选择。确认认证字段为 `ANTHROPIC_AUTH_TOKEN`，然后点击 **保存**。

<Frame>
  ![配置 Claude 模型映射](/images/image-11.png)
</Frame>

### 第五步：配置 Codex 模型

切换到顶部的 Codex 标签，添加 Xcompute 供应商，并使用带 `/v1` 的请求地址。

| 模型名称 | 模型 ID |
| --- | --- |
| GPT 5.6 Sol | `gpt-5.6-sol` |
| GPT 5.6 Terra | `gpt-5.6-terra` |
| GPT 5.6 Luna | `gpt-5.6-luna` |
| GPT 5.5 | `gpt-5.5` |
| GPT 5.4 | `gpt-5.4` |

### 第六步（Codex CLI 专属）：开启本地路由

Codex CLI 使用 OpenAI 协议，但如果你想通过 CC Switch 让 Codex 调用 Claude 等非 OpenAI 模型，需要开启本地路由做协议转换。

**操作步骤：**

1. 打开 CC Switch **设置** → **路由设置**
2. 开启 **路由总开关** ⚠️ 必须先开这个
3. 再开启 **Codex 路由开关**
4. 确认路由监听地址，通常是 `http://127.0.0.1:4142`

**环境变量配置：**

```bash
export OPENAI_BASE_URL="http://127.0.0.1:4142/v1"
export OPENAI_API_KEY="any-key-placeholder"
```

两个开关都要开，缺一不可。

## 测试连接

### 使用 CC Switch 测试

1. 打开刚创建的 Xcompute 供应商
2. 点击请求地址右侧的 **管理与测速**
3. 确认端点可以连接并查看响应延迟
4. 保存测试通过的请求地址

### 使用 curl 排查

如果连接失败，可以在终端直接检查 Xcompute 接口。将 `YOUR_API_KEY` 替换为你的真实密钥：

```bash
curl https://xcompute.us/v1/models \
  -H "Authorization: Bearer YOUR_API_KEY"
```

## 常见问题

### 如何切换回官方 API？

在 CC Switch 的预设供应商中添加官方供应商，切换过去后重新启动对应的 CLI 工具，并按官方登录或 OAuth 流程操作。

### 连接失败怎么办？

按以下顺序排查：

1. 确认 API Key 正确、未过期且账户有余额
2. 确认请求地址与工具匹配：Claude Code 不加 `/v1`，Codex 需要 `/v1`
3. 使用上面的 curl 命令测试网络和密钥
4. 在 CC Switch 中查看错误日志和测速结果
5. 国内网络可以切换到 `https://intl.dualseason.com`

### 如何同时管理多个工具？

CC Switch 支持 Claude Code、Claude Desktop、Codex、Gemini CLI、OpenCode、OpenClaw 和 Hermes Agent。在顶部工具栏分别选择工具并配置供应商即可。

### 响应很慢怎么办？

1. 切换到加速入口 `https://intl.dualseason.com`
2. 使用 **管理与测速** 选择延迟较低的端点
3. 检查本地网络和代理设置

### 配置备份在哪里？

- **数据库**：`~/.cc-switch/cc-switch.db`
- **本地设置**：`~/.cc-switch/settings.json`
- **备份位置**：`~/.cc-switch/backups/`
- **备份数量**：自动轮换并保留最近 10 个版本

可以在 CC Switch 设置中恢复备份。

## 高级功能

### 多 Key 轮换

如果你有多个 Xcompute API Key 分摊用量或走不同折扣渠道，可以添加多个 Key 并设置轮换策略：

- **轮询（Round Robin）**：按顺序轮流使用
- **随机（Random）**：随机选取
- **按权重**：根据权重比例分配

适用于不同 Key 属于不同渠道的场景（如一个官 Key 4.9 折、一个福利渠道 0.9 折）。

### 模型别名

在 CC Switch 中配置模型别名，快速切换：

| 别名 | 实际模型 | 用途 |
| --- | --- | --- |
| `claude-fast` | `claude-sonnet-5` | 日常编码，速度快 |
| `claude-smart` | `claude-fable-5` | 复杂任务，质量优先 |
| `gpt-cheap` | `gpt-5.4` | 简单任务，成本最低 |
