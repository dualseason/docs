使用 `gpt-image-2` 和参考图片生成新图片。请求必须使用 `multipart/form-data` 上传图片文件。

<Warning>
  使用 `curl -F` 或 SDK 的 multipart 功能时，不要手动设置 `Content-Type`。客户端需要自动生成包含 boundary 的请求头。
</Warning>

## 请求示例

<RequestExample>
  ```bash cURL theme={null}
  curl --request POST \
    --url https://xcompute.us/v1/images/edits \
    --header 'Authorization: Bearer YOUR_API_KEY' \
    --form 'model=gpt-image-2' \
    --form 'prompt=保留主体，将画面转换为清新的水彩插画' \
    --form 'image=@./reference.jpg' \
    --form 'n=1' \
    --form 'size=1024x1024'
  ```

  ```python Python theme={null}
  import requests

  with open("reference.jpg", "rb") as image:
      response = requests.post(
          "https://xcompute.us/v1/images/edits",
          headers={"Authorization": "Bearer YOUR_API_KEY"},
          data={
              "model": "gpt-image-2",
              "prompt": "保留主体，将画面转换为清新的水彩插画",
              "n": "1",
              "size": "1024x1024",
          },
          files={"image": ("reference.jpg", image, "image/jpeg")},
          timeout=300,
      )

  response.raise_for_status()
  print(response.json())
  ```

## 请求参数

<ParamField header="Authorization" type="string" required>
  Bearer API Key，例如 `Bearer sk-xxxxxxxx`。
</ParamField>

<ParamField body="model" type="string" default="gpt-image-2" required>
  固定填写 `gpt-image-2`。
</ParamField>

<ParamField body="prompt" type="string" required>
  描述希望如何参考或修改上传的图片，并说明哪些内容需要保留。
</ParamField>

<ParamField body="image" type="file" required>
  参考图片文件。推荐使用 JPEG 或 PNG。
</ParamField>

<ParamField body="mask" type="file">
  可选蒙版图片，用于局部重绘。建议与参考图片尺寸一致。
</ParamField>

<ParamField body="n" type="integer" default="1">
  生成图片数量。建议使用 `1`。
</ParamField>

<ParamField body="size" type="string" default="1024x1024">
  输出尺寸，例如 `1024x1024`。实际支持的尺寸取决于当前渠道。
</ParamField>

<ParamField body="quality" type="string">
  图片质量，例如 `auto`、`standard`、`high`。该参数可能因渠道不同而被忽略。
</ParamField>

<ParamField body="input_fidelity" type="string">
  输入图片保真度，可传 `low` 或 `high`。是否生效取决于当前渠道。
</ParamField>

<ParamField body="response_format" type="string">
  期望的返回格式，可传 `url` 或 `b64_json`。客户端应同时兼容两种实际返回形式。
</ParamField>
