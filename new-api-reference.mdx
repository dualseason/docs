这份文档按旧版接口页的写法整理新版接口。你可以把它当成新版的统一入口，按接口逐个查看请求、参数和返回。

## 基础信息

- `Base URL`：`https://xcompute.us`
- 控制台接口前缀：`/api`
- 模型调用接口前缀：`/v1`、`/v1beta`
- 控制台鉴权：登录后的 session，或用户 access token
- 模型调用鉴权：`Authorization: Bearer YOUR_API_KEY`

## 接口

<Tabs>
  <Tab title="GET">
    ### `GET /api/user/self`

    - 获取当前用户信息、余额和权限
    - 常用于控制台首页和个人中心

    <ParamField header="Authorization" type="string" required>
      登录态接口，使用当前 session 即可。
    </ParamField>

    <ResponseExample>
      ```json 200 theme={null}
      {
        "quota": 100,
        "used_quota": 20,
        "request_count": 12,
        "group": "default"
      }
      ```
    </ResponseExample>

    ### `GET /v1/models`

    - 获取当前 API Key 可用模型列表
    - 新项目优先使用这个接口

    <RequestExample>
      ```bash cURL theme={null}
      curl --request GET \
        --url https://xcompute.us/v1/models \
        --header 'Authorization: Bearer YOUR_API_KEY'
      ```
    </RequestExample>

    <ResponseExample>
      ```json 200 theme={null}
      {
        "success": true,
        "object": "list",
        "data": [
          {
            "id": "gpt-4o",
            "object": "model",
            "created": 1626777600,
            "owned_by": "openai"
          }
        ]
      }
      ```
    </ResponseExample>

    ### `GET /v1/tasks/{task_id}`

    - 查询异步任务状态
    - 常用于图像、视频和素材审核任务

    <ParamField path="task_id" type="string" required>
      任务 ID。
    </ParamField>

    <RequestExample>
      ```bash cURL theme={null}
      curl --request GET \
        --url 'https://xcompute.us/v1/tasks/YOUR_TASK_ID?language=zh' \
        --header 'Authorization: Bearer YOUR_API_KEY'
      ```
    </RequestExample>
  </Tab>

  <Tab title="POST">
    ### `POST /api/user/login`

    - 账号密码登录
    - 登录成功后写入 session
    - 如果账号启用 2FA，会继续返回二次验证流程

    <RequestExample>
      ```bash cURL theme={null}
      curl --request POST \
        --url https://xcompute.us/api/user/login \
        --header 'Content-Type: application/json' \
        --data '{
          "username": "YOUR_USERNAME",
          "password": "YOUR_PASSWORD"
        }'
      ```
    </RequestExample>

    ### `POST /v1/chat/completions`

    - OpenAI Chat Completions 兼容接口
    - 通过 `model` 参数选择模型

    <RequestExample>
      ```bash cURL theme={null}
      curl --request POST \
        --url https://xcompute.us/v1/chat/completions \
        --header 'Authorization: Bearer YOUR_API_KEY' \
        --header 'Content-Type: application/json' \
        --data '{
          "model": "gpt-4o",
          "messages": [
            {
              "role": "user",
              "content": "你好，请介绍一下你自己"
            }
          ]
        }'
      ```
    </RequestExample>

    <ParamField body="model" type="string" required>
      模型名，例如 `gpt-4o`。
    </ParamField>

    <ParamField body="messages" type="array" required>
      对话消息数组。
    </ParamField>

    ### `POST /v1/videos/generations`

    - 提交视频生成任务
    - 返回 `task_id` 后再查询状态

    <RequestExample>
      ```bash cURL theme={null}
      curl --request POST \
        --url https://xcompute.us/v1/videos/generations \
        --header 'Authorization: Bearer YOUR_API_KEY' \
        --header 'Content-Type: application/json' \
        --data '{
          "model": "grok-imagine-1.0-video-apimart",
          "prompt": "一只狗在海滩上奔跑，阳光明媚，慢镜头",
          "size": "16:9",
          "duration": 6,
          "quality": "720p"
        }'
      ```
    </RequestExample>

    <ParamField body="model" type="string" required>
      视频模型名。
    </ParamField>

    <ParamField body="prompt" type="string" required>
      视频内容描述，支持中英文。
    </ParamField>

    <ParamField body="duration" type="integer">
      视频时长，单位秒，必须传数字。
    </ParamField>
  </Tab>
</Tabs>
