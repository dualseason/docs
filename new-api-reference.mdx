这份文档整理当前新版 new-api 服务的主要接口。你可以优先按这里的接口接入新业务。

## 基础信息

| 项目 | 说明 |
| --- | --- |
| Base URL | `https://xcompute.us` |
| 控制台接口前缀 | `/api` |
| 模型调用接口前缀 | `/v1`、`/v1beta` |
| 控制台鉴权 | 登录后的 session，或用户 access token |
| 模型调用鉴权 | `Authorization: Bearer sk-...` |

控制台接口主要给网页后台和用户中心使用。模型调用接口主要给代码、第三方客户端和 AI 工具使用。

## 鉴权方式

### 控制台登录态

你可以先调用登录接口，服务端会写入 session。后续访问 `/api/user/self`、`/api/token/` 等控制台接口时复用登录态。

```bash
curl --request POST \
  --url https://xcompute.us/api/user/login \
  --header 'Content-Type: application/json' \
  --data '{
    "username": "YOUR_USERNAME",
    "password": "YOUR_PASSWORD"
  }'
```

登录成功返回用户基础信息。如果账号启用了 2FA，会返回 `require_2fa: true`，需要继续调用 2FA 验证接口。

### API Key 鉴权

模型调用接口使用 API Key。

```http
Authorization: Bearer YOUR_API_KEY
```

API Key 一般用于 `/v1/chat/completions`、`/v1/models`、`/v1/videos/generations` 等接口。

## 登录与账号接口

| 方法 | 路径 | 鉴权 | 用途 |
| --- | --- | --- | --- |
| `POST` | `/api/user/register` | 无 | 注册账号 |
| `POST` | `/api/user/login` | 无 | 账号密码登录 |
| `POST` | `/api/user/login/2fa` | 临时登录态 | 完成 2FA 登录 |
| `POST` | `/api/user/passkey/login/begin` | 无 | 开始 Passkey 登录 |
| `POST` | `/api/user/passkey/login/finish` | 无 | 完成 Passkey 登录 |
| `GET` | `/api/user/logout` | 登录态 | 退出登录 |
| `GET` | `/api/user/self` | 登录态 | 获取当前用户信息、余额和权限 |
| `PUT` | `/api/user/self` | 登录态 | 更新当前用户资料或偏好 |
| `DELETE` | `/api/user/self` | 登录态 | 删除当前用户 |
| `GET` | `/api/user/token` | 登录态 | 生成用户 access token |

`/api/user/self` 是前端最常用的用户信息接口。它会返回 `quota`、`used_quota`、`request_count`、`group`、`role`、`permissions` 等字段。

```json
  ]
}
```

## 余额与用量接口

### 用户余额

| 方法 | 路径 | 鉴权 | 用途 |
| --- | --- | --- | --- |
| `GET` | `/api/user/self` | 登录态 | 获取用户余额、已用额度和请求次数 |
| `GET` | `/api/data/self` | 登录态 | 获取当前用户额度消耗统计 |
| `GET` | `/api/data/flow/self` | 登录态 | 获取当前用户流量消耗统计 |

`/api/user/self` 返回的关键字段：

| 字段 | 说明 |
| --- | --- |
| `quota` | 当前剩余额度 |
| `used_quota` | 已使用额度 |
| `request_count` | 请求次数 |
| `group` | 当前用户分组 |

### API Key 用量

| 方法 | 路径 | 鉴权 | 用途 |
| --- | --- | --- | --- |
| `GET` | `/api/usage/token/` | API Key | 查询当前 API Key 的额度和限制 |

示例：

```bash
curl --request GET \
  --url https://xcompute.us/api/usage/token/ \
  --header 'Authorization: Bearer YOUR_API_KEY'
```

返回的 `data` 包含 `total_granted`、`total_used`、`total_available`、`unlimited_quota`、`model_limits`、`expires_at` 等字段。
